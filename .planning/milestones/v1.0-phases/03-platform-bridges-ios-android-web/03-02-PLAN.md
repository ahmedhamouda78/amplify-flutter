  <files>packages/auth/amplify_auth_cognito/darwin/Classes/WebAuthnBridgeImpl.swift</files>
  <action>
    Create `darwin/Classes/WebAuthnBridgeImpl.swift` implementing the Pigeon-generated `WebAuthnBridgeApi` protocol.

    **Class structure:**
    ```swift
    import AuthenticationServices
    import Foundation
    #if os(iOS)
    import Flutter
    import UIKit
    #elseif os(macOS)
    import FlutterMacOS
    #endif

    class WebAuthnBridgeImpl: NSObject, WebAuthnBridgeApi { ... }
    ```

    **isPasskeySupported:**
    - Use `if #available(iOS 17.4, macOS 13.5, *)` runtime check (per locked decision -- runtime API availability, not OS version)
    - Return `true` if available, `false` otherwise

    **createCredential(optionsJson:completion:):**
    - Guard `#available(iOS 17.4, macOS 13.5, *)`, else return FlutterError(code: "notSupported")
    - Parse `optionsJson` via JSONSerialization to extract:
      - `challenge` (base64url string -> Data)
      - `rp.id` (String)
      - `user.id` (base64url string -> Data)
      - `user.name` (String)
      - `user.displayName` (String, optional)
      - `pubKeyCredParams` (array, optional -- for excludeCredentials)
      - `authenticatorSelection` (optional)
      - `attestation` (optional)
      - `excludeCredentials` (optional array of {id, type, transports})
    - Create `ASAuthorizationPlatformPublicKeyCredentialProvider(relyingPartyIdentifier: rpId)`
    - Build registration request via `provider.createCredentialRegistrationRequest(challenge:name:userID:)`
    - If `excludeCredentials` present, set `request.excludedCredentials` (map each to `ASAuthorizationPlatformPublicKeyCredentialDescriptor`)
    - Create `ASAuthorizationController(authorizationRequests: [request])`
    - **CRITICAL:** Store controller and delegate as instance properties to prevent ARC deallocation (see Research pitfall #1)
    - Set `presentationContextProvider` to self (implement `ASAuthorizationControllerPresentationContextProviding`)
    - On success (delegate callback): serialize response to JSON with:
      - `id`: base64url encode of `credentialID`
      - `rawId`: base64url encode of `credentialID`
      - `type`: "public-key"
      - `response.clientDataJSON`: base64url encode
      - `response.attestationObject`: base64url encode
      - `authenticatorAttachment`: "platform"
    - On failure: map `ASAuthorizationError.Code` to FlutterError codes:
      - `.canceled` -> "cancelled"
      - `.failed`, `.invalidResponse`, `.unknown` -> "registrationFailed"
      - `.notHandled` -> "notSupported"
      - `.matchedExcludedCredential` -> "registrationFailed"

    **getCredential(optionsJson:completion:):**
    - Guard `#available(iOS 17.4, macOS 13.5, *)`, else FlutterError "notSupported"
    - Parse JSON for: `challenge`, `rpId`, `allowCredentials` (optional), `userVerification` (optional)
    - Create assertion request via `provider.createCredentialAssertionRequest(challenge:)`
    - If `allowCredentials` present, set `request.allowedCredentials` (map each to descriptor)
    - Execute with ASAuthorizationController (same delegate pattern)
    - On success: serialize response JSON:
      - `id`: base64url of credentialID
      - `rawId`: base64url of credentialID
      - `type`: "public-key"
      - `response.clientDataJSON`: base64url
      - `response.authenticatorData`: base64url
      - `response.signature`: base64url
      - `response.userHandle`: base64url (if present)
      - `authenticatorAttachment`: "platform"
    - On failure: map to error codes (same as create, but default to "assertionFailed")

    **Base64url helpers:**
    Add a private `Data` extension for base64url encoding/decoding:
    - `init?(base64urlEncoded:)` -- convert base64url to base64 (replace `-`/`_` with `+`/`/`, add padding), then decode
    - `base64urlEncodedString()` -- encode to base64, then replace `+`/`/` with `-`/`_`, remove padding

    **ASAuthorizationControllerPresentationContextProviding:**
    Implement to return the key window:
    ```swift
    #if os(iOS)
    // Use UIApplication.shared.connectedScenes to find the key window (modern iOS)
    #elseif os(macOS)
    // Use NSApplication.shared.keyWindow
    #endif
    ```

    **Delegate pattern:**
    Use an inner class or closure-based approach. Store a pending completion handler and the controller reference as instance properties. The delegate callbacks (`didCompleteWithAuthorization`, `didCompleteWithError`) call the stored completion and nil out the references.
