    **isPasskeySupported():**
    - Check `web.window.navigator` has `credentials` property
    - Check `PublicKeyCredential` exists in the global scope via JS interop (e.g., `globalContext.has('PublicKeyCredential'.toJS)`)
    - Per locked decision: API existence check only, NOT `isUserVerifyingPlatformAuthenticatorAvailable()`
    - Return `true` if both exist, `false` otherwise

    **createCredential(optionsJson):**
    1. Parse `optionsJson` via `json.decode` to get `Map<String, dynamic>`
    2. Extract `publicKey` options (the JSON may be the publicKey options directly or wrapped)
    3. Convert base64url string fields to JS ArrayBuffer:
       - `challenge` -> ArrayBuffer (REQUIRED)
       - `user.id` -> ArrayBuffer (REQUIRED)
       - `excludeCredentials[].id` -> ArrayBuffer (each, if present)
    4. Build JS options object using `dart:js_interop` -- construct `{ publicKey: { ... } }` shape
       Use `JSObject()` with property assignment or `js_util`-style construction.
       The exact approach: create a JS object literal and set properties. Use extension types from `package:web` where available (`CredentialCreationOptions`, `PublicKeyCredentialCreationOptions`), or construct manually.
    5. Call `web.window.navigator.credentials.create(options)` which returns `Future<web.Credential?>`
    6. Cast result to `web.PublicKeyCredential`
    7. Extract response as `web.AuthenticatorAttestationResponse`:
       - `response.clientDataJSON` -> ArrayBuffer -> base64url string
       - `response.attestationObject` -> ArrayBuffer -> base64url string
    8. Build JSON response:
       ```json
       {
         "id": "<base64url of rawId>",
         "rawId": "<base64url of rawId>",
         "type": "public-key",
         "response": {
           "clientDataJSON": "<base64url>",
           "attestationObject": "<base64url>"
         },
         "authenticatorAttachment": "platform"
       }
       ```
    9. Return `json.encode(response)`
    10. Wrap in try/catch for JS exceptions:
        - Catch `JSObject` errors (DOMException) and map per the error table in interfaces
        - Access `name` property of DOMException to determine type

    **getCredential(optionsJson):**
    1. Parse JSON, convert base64url fields to ArrayBuffer:
       - `challenge` -> ArrayBuffer (REQUIRED)
       - `allowCredentials[].id` -> ArrayBuffer (each, if present)
    2. Build `{ publicKey: { ... } }` options object
    3. Call `web.window.navigator.credentials.get(options)`
    4. Cast to `web.PublicKeyCredential`, get response as `web.AuthenticatorAssertionResponse`:
       - `response.clientDataJSON` -> base64url
       - `response.authenticatorData` -> base64url
       - `response.signature` -> base64url
       - `response.userHandle` -> base64url (if non-null)
    5. Build JSON response:
       ```json
       {
         "id": "<base64url of rawId>",
         "rawId": "<base64url of rawId>",
         "type": "public-key",
         "response": {
           "clientDataJSON": "<base64url>",
           "authenticatorData": "<base64url>",
           "signature": "<base64url>",
           "userHandle": "<base64url or null>"
         },
         "authenticatorAttachment": "platform"
       }
       ```
    6. Error handling same as createCredential but defaults to PasskeyAssertionFailedException

    **Helper functions (private):**
    - `JSArrayBuffer _base64urlToArrayBuffer(String base64url)` -- decode base64url to bytes, convert to JSArrayBuffer
    - `String _arrayBufferToBase64url(JSArrayBuffer buffer)` -- convert JSArrayBuffer to Dart bytes, encode as base64url
    - `PasskeyException _mapDomException(Object error, {required bool isCreate})` -- inspect JS error for DOMException name and map to PasskeyException subtypes

    **IMPORTANT (from research pitfall #4):** The browser `navigator.credentials.create()` expects `ArrayBuffer` for `challenge`, `user.id`, etc. -- NOT strings. Must convert base64url string fields to `JSArrayBuffer` before passing to the API. The `package:web` types should provide `CredentialCreationOptions` and `PublicKeyCredentialCreationOptions` but they may not have all fields typed. Use `jsify()` or manual JS object construction as needed.

    **IMPORTANT (from research pitfall #3):** Response fields like `clientDataJSON`, `attestationObject`, `authenticatorData`, `signature` come back as `ArrayBuffer` from the browser. Must convert to base64url strings via `_arrayBufferToBase64url()`.
