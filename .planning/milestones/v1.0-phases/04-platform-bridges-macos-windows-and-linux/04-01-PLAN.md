  <name>Task 1: Create Windows WebAuthn FFI bindings</name>
  <files>packages/auth/amplify_auth_cognito/lib/src/windows/webauthn_bindings.dart</files>
  <action>
Create `webauthn_bindings.dart` with raw `dart:ffi` bindings to `webauthn.dll`. This file defines all FFI typedefs, struct definitions, and function lookups needed by the platform implementation.

**Load the DLL:**
```dart
final _webauthn = DynamicLibrary.open('webauthn.dll');
```

**Functions to bind (with Dart-friendly wrappers):**

1. `WebAuthNGetApiVersionNumber() -> int` — returns API version number
2. `WebAuthNIsUserVerifyingPlatformAuthenticatorAvailable(Pointer<Int32>) -> int` — HRESULT, writes bool to pointer
3. `WebAuthNAuthenticatorMakeCredential(IntPtr hWnd, Pointer rpInfo, Pointer userInfo, Pointer pubKeyCredParams, Pointer clientData, Pointer options, Pointer<Pointer> ppResult) -> int` — HRESULT
4. `WebAuthNGetAssertion(IntPtr hWnd, Pointer<Utf16> rpId, Pointer clientData, Pointer options, Pointer<Pointer> ppResult) -> int` — HRESULT
5. `WebAuthNFreeCredentialAttestation(Pointer) -> void`
6. `WebAuthNFreeAssertion(Pointer) -> void`
7. `GetActiveWindow() -> IntPtr` (from user32.dll for HWND acquisition)

**Struct definitions (use `Struct` subclasses):**

For JSON pass-through (API version 4+), the key structs are minimal:
- `WebAuthnRpEntityInformation` (dwVersion, pwszId, pwszName, pwszIcon)
- `WebAuthnUserEntityInformation` (dwVersion, cbId, pbId, pwszName, pwszIcon, pwszDisplayName)
- `WebAuthnCoseCredentialParameter` (dwVersion, pwszCredentialType, lAlg)
- `WebAuthnCoseCredentialParameters` (cCredentialParameters, pCredentialParameters)
- `WebAuthnClientData` (dwVersion, cbClientDataJSON, pbClientDataJSON, pwszHashAlgId)
- `WebAuthnAuthenticatorMakeCredentialOptions` — version 5+ struct with the JSON pass-through fields: `cbPublicKeyCredentialCreationOptionsJSON` and `pbPublicKeyCredentialCreationOptionsJSON` at the end
- `WebAuthnAuthenticatorGetAssertionOptions` — version 6+ struct with JSON pass-through fields
- `WebAuthnCredentialAttestation` — result struct with `cbRegistrationResponseJSON` / `pbRegistrationResponseJSON` (version 4+)
- `WebAuthnAssertion` — result struct with `cbAuthenticationResponseJSON` / `pbAuthenticationResponseJSON` (version 3+)

**Constants:**
- API version constants: `WEBAUTHN_API_VERSION_4 = 4`
- HRESULT success: `S_OK = 0`
- Key HRESULT error codes: `NTE_USER_CANCELLED = 0x80090036`, `NTE_NOT_FOUND = 0x80090011`, `NTE_INVALID_PARAMETER = 0x80090027`
- `WEBAUTHN_API_CURRENT_VERSION` for the options struct version field

Wrap all lookups in a `WebAuthnBindings` class with lazy final fields so the class can be injected for testing. Constructor takes optional `DynamicLibrary` parameters (defaults to `DynamicLibrary.open('webauthn.dll')` and `DynamicLibrary.open('user32.dll')`).

Use `package:ffi` for `Arena`, `calloc`, `Utf16` extensions. Import `dart:ffi` for core FFI types.

**IMPORTANT:** The struct layouts for the MakeCredential options and GetAssertion options are large and version-dependent. Focus on the fields needed for JSON pass-through mode (version 4+). For the options structs, allocate them as raw `Pointer<Uint8>` with `calloc<Uint8>(structSize)` and write fields at known offsets, rather than trying to define the full struct hierarchy in Dart. This avoids the complexity of correctly aligning 20+ fields including nested pointers.

<task type="auto">
  <name>Task 2: Create WindowsWebAuthnPlatform implementation</name>
  <files>packages/auth/amplify_auth_cognito/lib/src/windows/windows_webauthn_platform.dart</files>
  <action>
Create `windows_webauthn_platform.dart` implementing `WebAuthnCredentialPlatform` using the FFI bindings from Task 1.

**Constructor:**
- Accept optional `WebAuthnBindings` parameter for testability (defaults to `WebAuthnBindings()`)
- Check API version on construction; store it for later use

**isPasskeySupported():**
- Try to load webauthn.dll (handle DLL not found gracefully)
- If API version < 4, return false (too old for JSON pass-through)
- Call `WebAuthNIsUserVerifyingPlatformAuthenticatorAvailable` — if HRESULT is S_OK and value is true, return true
- Return false on any error

**createCredential(String optionsJson):**
1. Get HWND via `GetActiveWindow()` — throw if NULL (no active window)
2. Parse minimal fields from optionsJson: `rp.id`, `rp.name`, `user.id`, `user.name`, `user.displayName`, `pubKeyCredParams` (for the required struct fields)
3. Use Arena for memory management
4. Populate the minimal required structs (RP info, user info, credential params, client data)
5. For JSON pass-through: set `pbPublicKeyCredentialCreationOptionsJSON` to the raw optionsJson bytes (UTF-8 encoded) and `cbPublicKeyCredentialCreationOptionsJSON` to its length
6. Call `WebAuthNAuthenticatorMakeCredential`
7. Check HRESULT — map errors via `_mapHResult`
8. Read `pbRegistrationResponseJSON` from the result attestation struct (UTF-8 string)
9. Free the attestation with `WebAuthNFreeCredentialAttestation` in a finally block
10. Return the JSON response string

**getCredential(String optionsJson):**
1. Get HWND via `GetActiveWindow()`
2. Parse `rpId` from optionsJson
3. For JSON pass-through: set `pbPublicKeyCredentialRequestOptionsJSON` to raw optionsJson bytes
4. Call `WebAuthNGetAssertion`
5. Check HRESULT — map errors
6. Read `pbAuthenticationResponseJSON` from assertion result
7. Free with `WebAuthNFreeAssertion` in finally block
8. Return JSON response

**Error mapping (`_mapHResult`):**
- Accept `int hr` and `bool isRegistration` parameters
- `0x80090036` (NTE_USER_CANCELLED) -> `PasskeyCancelledException`
- `0x80090011` (NTE_NOT_FOUND) -> `PasskeyAssertionFailedException`
- `0x80090027` (NTE_INVALID_PARAMETER) -> registration ? `PasskeyRegistrationFailedException` : `PasskeyAssertionFailedException`
- Default -> registration ? `PasskeyRegistrationFailedException` : `PasskeyAssertionFailedException`
- Include hex HRESULT in error message for debugging: `'Windows WebAuthn error: 0x${hr.toRadixString(16)}'`

**Memory management:** Use `using((Arena arena) { ... })` pattern from `package:ffi` to ensure all allocated memory is freed. For the result structs returned by the API, always free with the corresponding WebAuthN free function in a finally block outside the Arena.
