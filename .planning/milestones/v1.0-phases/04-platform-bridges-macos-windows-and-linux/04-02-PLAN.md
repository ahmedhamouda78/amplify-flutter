  <action>
Create `libfido2_bindings.dart` with `dart:ffi` bindings to `libfido2.so`. This file defines all FFI typedefs, function lookups, and error constants for the libfido2 C API.

**LibFido2Bindings class:**
- Constructor takes a `DynamicLibrary` (loaded externally — the platform class handles dynamic loading)
- All function lookups are `late final` fields

**Functions to bind:**

Device discovery:
- `fido_dev_info_new(int n) -> Pointer` — allocate device info list
- `fido_dev_info_free(Pointer *devlist, int n) -> void` — free device info list
- `fido_dev_info_manifest(Pointer devlist, int n, Pointer<IntPtr> found) -> int` — discover devices
- `fido_dev_info_path(Pointer di) -> Pointer<Utf8>` — get device path from info entry
- `fido_dev_info_ptr(Pointer devlist, int idx) -> Pointer` — get entry at index

Device management:
- `fido_dev_new() -> Pointer` — allocate device handle
- `fido_dev_free(Pointer *dev) -> void` — free device
- `fido_dev_open(Pointer dev, Pointer<Utf8> path) -> int` — open device
- `fido_dev_close(Pointer dev) -> int` — close device

Credential creation (registration):
- `fido_cred_new() -> Pointer` — allocate credential
- `fido_cred_free(Pointer *cred) -> void` — free credential
- `fido_cred_set_type(Pointer cred, int type) -> int` — set algorithm (e.g., COSE_ES256 = -7)
- `fido_cred_set_clientdata_hash(Pointer cred, Pointer<Uint8> hash, int hashLen) -> int`
- `fido_cred_set_rp(Pointer cred, Pointer<Utf8> rpId, Pointer<Utf8> rpName) -> int`
- `fido_cred_set_user(Pointer cred, Pointer<Uint8> userId, int userIdLen, Pointer<Utf8> userName, Pointer<Utf8> displayName, Pointer<Utf8> icon) -> int`
- `fido_cred_set_rk(Pointer cred, int rk) -> int` — set resident key requirement
- `fido_cred_set_uv(Pointer cred, int uv) -> int` — set user verification
- `fido_dev_make_cred(Pointer dev, Pointer cred, Pointer<Utf8> pin) -> int` — perform ceremony

Credential result getters:
- `fido_cred_id_ptr(Pointer cred) -> Pointer<Uint8>` + `fido_cred_id_len(Pointer cred) -> int`
- `fido_cred_authdata_ptr(Pointer cred) -> Pointer<Uint8>` + `fido_cred_authdata_len(Pointer cred) -> int`
- `fido_cred_x5c_ptr(Pointer cred) -> Pointer<Uint8>` + `fido_cred_x5c_len(Pointer cred) -> int`
- `fido_cred_sig_ptr(Pointer cred) -> Pointer<Uint8>` + `fido_cred_sig_len(Pointer cred) -> int`
- `fido_cred_fmt(Pointer cred) -> Pointer<Utf8>` — attestation format
- `fido_cred_clientdata_hash_ptr(Pointer cred) -> Pointer<Uint8>` + len

Assertion (sign-in):
- `fido_assert_new() -> Pointer` — allocate assertion
- `fido_assert_free(Pointer *assert) -> void` — free assertion
- `fido_assert_set_clientdata_hash(Pointer assert, Pointer<Uint8> hash, int len) -> int`
- `fido_assert_set_rp(Pointer assert, Pointer<Utf8> rpId) -> int`
- `fido_assert_set_uv(Pointer assert, int uv) -> int`
- `fido_assert_allow_cred(Pointer assert, Pointer<Uint8> credId, int len) -> int` — add allowed credential
- `fido_dev_get_assert(Pointer dev, Pointer assert, Pointer<Utf8> pin) -> int` — perform ceremony
- `fido_assert_count(Pointer assert) -> int` — number of assertions returned

Assertion result getters (take `idx` parameter for which assertion):
- `fido_assert_authdata_ptr(Pointer assert, int idx) -> Pointer<Uint8>` + `fido_assert_authdata_len`
- `fido_assert_sig_ptr(Pointer assert, int idx) -> Pointer<Uint8>` + `fido_assert_sig_len`
- `fido_assert_user_id_ptr(Pointer assert, int idx) -> Pointer<Uint8>` + `fido_assert_user_id_len`
- `fido_assert_id_ptr(Pointer assert, int idx) -> Pointer<Uint8>` + `fido_assert_id_len`

Init:
- `fido_init(int flags) -> void` — initialize libfido2 (call with 0)

**Error constants:**

static const int FIDO_ERR_TX = -1;  // transmission error

<task type="auto">
  <name>Task 2: Create LinuxWebAuthnPlatform implementation</name>
  <files>packages/auth/amplify_auth_cognito/lib/src/linux/linux_webauthn_platform.dart</files>
  <action>
Create `linux_webauthn_platform.dart` implementing `WebAuthnCredentialPlatform` using the libfido2 FFI bindings from Task 1.

**Constructor:**
- Try `DynamicLibrary.open('libfido2.so')` — catch `ArgumentError`, store null if unavailable
- If loaded, create `LibFido2Bindings` and call `fido_init(0)` to initialize the library
- Accept optional `LibFido2Bindings?` parameter for testability

**isPasskeySupported():**
- Return `_bindings != null` (library loaded successfully)
- No device discovery here — just library availability

**Helper: _ensureSupported():**
- If `_bindings == null`, throw `PasskeyNotSupportedException('libfido2 is not available. Install libfido2-dev for passkey support.')`

**Helper: _discoverAndOpenDevice():**
1. Call `fido_dev_info_manifest` with max 64 entries
2. If found count is 0, throw `PasskeyNotSupportedException('No FIDO2 security key detected. Please insert a FIDO2 USB key.')`
3. Get path of first device via `fido_dev_info_path(fido_dev_info_ptr(list, 0))`
4. Open device with `fido_dev_open`
5. Return device pointer (caller must close and free)

**Helper: _buildClientDataJsonAndHash(String type, String challenge, String origin):**
1. Construct clientDataJSON: `{"type":"$type","challenge":"$challenge","origin":"$origin","crossOrigin":false}`
2. SHA-256 hash it (use `dart:convert` + `package:crypto` or compute manually — check if project already has crypto dependency; if not, use `dart:typed_data` with a simple SHA-256 from the Dart SDK)
3. Return both the raw clientDataJSON bytes and the hash bytes

Actually — check if `package:crypto` is available in the project. If not, consider using the `Digest` from `package:aws_common` or compute SHA-256 via the platform's existing crypto utilities. The project likely has SHA-256 available through AWS SDK dependencies. Search for existing SHA-256 usage patterns in the codebase and reuse them. If nothing is readily available, add `package:crypto` to the pubspec dependencies.

**Helper: _parseUserVerification(String? uv):**
- Map Cognito's userVerification preference to libfido2 constants:
  - `'required'` -> `FIDO_OPT_TRUE`
  - `'preferred'` -> `FIDO_OPT_TRUE` (best effort)
  - `'discouraged'` -> `FIDO_OPT_FALSE`
  - null/default -> `FIDO_OPT_OMIT`

**createCredential(String optionsJson):**
1. `_ensureSupported()`
2. Parse optionsJson to extract: `rp.id`, `rp.name`, `user.id` (base64url), `user.name`, `user.displayName`, `pubKeyCredParams[0].alg`, `challenge` (base64url), `authenticatorSelection.residentKey`, `authenticatorSelection.userVerification`, `excludeCredentials`
3. Build clientDataJSON and hash for type `"webauthn.create"`
4. Discover and open device
5. Use `using((Arena arena) { ... })` for memory management
6. Create credential object, set all fields via fido_cred_set_* functions
7. Call `fido_dev_make_cred(dev, cred, nullptr)` (null PIN — rely on UV)
8. Check return code, map errors via `_mapLibFido2Error`
9. Read result fields: credential ID, authenticator data, attestation object components
10. **Assemble response JSON** manually (libfido2 does NOT provide JSON):
```json
{
  "id": "<base64url credential ID>",
  "rawId": "<base64url credential ID>",
  "type": "public-key",
  "response": {
    "clientDataJSON": "<base64url clientDataJSON>",
    "attestationObject": "<base64url attestation object bytes from fido_cred_authdata>",
    "transports": ["usb"]
  },
  "authenticatorAttachment": "cross-platform"
}
```
11. Close and free device, free credential in finally blocks
12. Return assembled JSON string

**getCredential(String optionsJson):**
1. `_ensureSupported()`
2. Parse: `rpId`, `challenge` (base64url), `allowCredentials[].id` (base64url), `userVerification`
3. Build clientDataJSON and hash for type `"webauthn.get"`
4. Discover and open device
5. Create assertion, set fields, add allowed credentials
6. Call `fido_dev_get_assert(dev, assert, nullptr)`
7. Map errors
8. Read result: authenticator data, signature, user handle, credential ID
9. **Assemble response JSON:**
```json
{
  "id": "<base64url credential ID>",
  "rawId": "<base64url credential ID>",
  "type": "public-key",
  "response": {
    "clientDataJSON": "<base64url clientDataJSON>",
    "authenticatorData": "<base64url authenticator data>",
    "signature": "<base64url signature>",
    "userHandle": "<base64url user handle or null>"
  },
  "authenticatorAttachment": "cross-platform"
}
```
10. Cleanup in finally blocks
11. Return JSON string

**Error mapping (_mapLibFido2Error):**
- `FIDO_ERR_NOT_ALLOWED` -> `PasskeyCancelledException`
- `FIDO_ERR_ACTION_TIMEOUT` -> `PasskeyCancelledException('Operation timed out')`
- `FIDO_ERR_PIN_REQUIRED` -> `PasskeyAssertionFailedException('PIN required but not provided')`
- `FIDO_ERR_UV_BLOCKED` -> `PasskeyAssertionFailedException('User verification blocked')`
- Default -> `isRegistration ? PasskeyRegistrationFailedException : PasskeyAssertionFailedException` with `'libfido2 error code: $errorCode'`

**Base64url helpers:** Use `dart:convert` base64Url encoder/decoder (strip padding for encoding, add padding for decoding). Check if the project's existing `webauthn_types.dart` has base64url utilities and reuse them.
