```dart

```

To:
```dart
if (zIsWeb) {
  return;
}

// Windows and Linux use direct FFI bridges (no Pigeon)
if (Platform.isWindows) {
  stateMachine.addInstance<WebAuthnCredentialPlatform>(
    WindowsWebAuthnPlatform(),
  );
  return;
}
if (Platform.isLinux) {
  stateMachine.addInstance<WebAuthnCredentialPlatform>(
    LinuxWebAuthnPlatform(),
  );
  return;
}

// iOS, Android, and macOS use the Pigeon bridge
if (!(Platform.isAndroid || Platform.isIOS || Platform.isMacOS)) {
  return;
}
```

The rest of the method body (NativeAuthPlugin setup, NativeAuthBridge, WebAuthnBridgeApi, etc.) remains unchanged — it now runs for iOS, Android, AND macOS (previously only iOS and Android).

**Key design points (per user decision):**
- macOS shares the exact same Pigeon registration path as iOS/Android — just add `Platform.isMacOS` to the guard
- Windows and Linux register FFI-based implementations directly, bypassing Pigeon
- Web early-returns before any platform checks (existing behavior)
- The existing `if (!zIsWeb && (Platform.isAndroid || Platform.isIOS))` guard later in the file (around line 107 for NativeAuthBridge in configure()) should also be updated to include `Platform.isMacOS`

**Also check for any other platform guards in the file** that reference `Platform.isAndroid || Platform.isIOS` and add `Platform.isMacOS` where the Pigeon bridge/native plugin applies. The NativeAuthBridge and NativeAuthPlugin are shared Darwin so macOS needs them too.

**Podspec update:**

In `amplify_auth_cognito.podspec`, change the macOS deployment target from `'10.15'` to `'13.5'` to support `ASAuthorizationController` for passkeys:

```ruby
# BEFORE:
s.osx.deployment_target = '10.15'

# AFTER:
s.osx.deployment_target = '13.5'
```

The Swift code already has `#available(macOS 13.5, *)` runtime guards in `WebAuthnBridgeImpl.swift`, so `isPasskeySupported()` will return false on older macOS versions at runtime. However, the deployment target must be high enough for the `ASAuthorizationPlatformPublicKeyCredentialProvider` API to be available at compile time.

Note: Per research pitfall, raising to 13.5 means the entire plugin requires macOS 13.5+. This is acceptable because macOS 13.5 (Ventura) is well-established and the plugin already targets iOS 13.0+ and Swift 5.9.

Also add `AuthenticationServices` to the macOS frameworks list since the WebAuthn bridge uses it:
```ruby
# BEFORE:
s.osx.frameworks = 'AppKit', 'IOKit'

# AFTER:
s.osx.frameworks = 'AppKit', 'IOKit', 'AuthenticationServices'
```
