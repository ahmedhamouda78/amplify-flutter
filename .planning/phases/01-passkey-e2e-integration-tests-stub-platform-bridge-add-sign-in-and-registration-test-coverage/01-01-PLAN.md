3. Override `addPlugin` to inject the mock BEFORE calling super (which sets up native bridges). The critical insight: calling `super.addPlugin()` from `AmplifyAuthCognito` will OVERWRITE our mock with the real platform bridge. So when `webAuthnPlatform` is non-null, we must call `AmplifyAuthCognitoDart`'s `addPlugin` directly to skip native bridge setup, then inject our mock:
   ```dart
   @override
   Future<void> addPlugin({
     required AmplifyAuthProviderRepository authProviderRepo,
   }) async {
     if (webAuthnPlatform != null) {
       // Skip AmplifyAuthCognito.addPlugin() which overwrites WebAuthn platform
       // with native bridge. Call AmplifyAuthCognitoDart.addPlugin() directly.
       await AmplifyAuthCognitoDart(
         secureStorageFactory: AmplifySecureStorage.factoryFrom(
           macOSOptions: MacOSSecureStorageOptions(useDataProtection: false),
         ),
       ).addPlugin(authProviderRepo: authProviderRepo);
       stateMachine.addInstance<WebAuthnCredentialPlatform>(webAuthnPlatform!);
     } else {
       await super.addPlugin(authProviderRepo: authProviderRepo);
     }
   }
   ```

   NOTE: This approach has a subtle issue -- `AmplifyAuthCognitoDart(...).addPlugin()` creates a NEW instance rather than calling super on the SAME instance. The correct approach is to use the parent's addPlugin. Since `AmplifyAuthCognito extends AmplifyAuthCognitoDart`, and `AmplifyAuthCognitoDart.addPlugin()` is what we want (it sets up the state machine without platform bridges), we need:
   ```dart
   // Call grandparent's addPlugin to skip platform bridge setup
   await super.addPlugin(authProviderRepo: authProviderRepo);
   // Then overwrite the platform bridge that super just set
   stateMachine.addInstance<WebAuthnCredentialPlatform>(webAuthnPlatform!);
   ```

   Actually, reviewing `AmplifyAuthCognito.addPlugin()` more carefully: it calls `super.addPlugin()` first (which is `AmplifyAuthCognitoDart.addPlugin()`), then adds platform-specific bridges. So our injection must happen AFTER `super.addPlugin()` to overwrite the platform bridge. The correct implementation:
   ```dart
   @override
   Future<void> addPlugin({
     required AmplifyAuthProviderRepository authProviderRepo,
   }) async {
     await super.addPlugin(authProviderRepo: authProviderRepo);
     if (webAuthnPlatform != null) {
       stateMachine.addInstance<WebAuthnCredentialPlatform>(webAuthnPlatform!);
     }
   }
   ```
   This works because `addInstance` replaces any existing instance for that type.

4. Add required additional imports at the top:
   ```dart
   import 'package:amplify_auth_cognito_dart/amplify_auth_cognito_dart.dart' show AmplifyAuthProviderRepository;
   ```
   Check if `AmplifyAuthProviderRepository` is already available via existing imports. It comes from `amplify_core` which is re-exported by `amplify_flutter`. If the existing imports already bring it in, no new import is needed. The `addPlugin` override signature must match the parent exactly.

    createCredential: (_) async => testRegistrationResponse,
    isPasskeySupported: () async => true,

/// Creates a [MockWebAuthnCredentialPlatform] that throws
/// [PasskeyCancelledException] when the user cancels.
MockWebAuthnCredentialPlatform createCancelledMockWebAuthnPlatform() {
  return MockWebAuthnCredentialPlatform(
    getCredential: (_) async =>
        throw const PasskeyCancelledException('User cancelled'),
    createCredential: (_) async =>
        throw const PasskeyCancelledException('User cancelled'),
    isPasskeySupported: () async => true,
  );
}

/// Creates a [MockWebAuthnCredentialPlatform] that throws

        throw const PasskeyNotSupportedException('Passkeys not supported'),
    createCredential: (_) async =>
        throw const PasskeyNotSupportedException('Passkeys not supported'),
    isPasskeySupported: () async => false,
